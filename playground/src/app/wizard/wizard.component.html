Welcome to notonly.rest

Get a not only RESTfull API in seconds. Full flavored with websockets, authentication and fake data.

Fragment

Fragments are the basic building blocks to define your data. It works like tagging: You simply add a fragment like #_customer: {{ '{}' }} to your data and you can recover it via the /customer endpoint.

Authentication

Reading by default is always allowed. Changing and Deleting however is secured by default and needs authentication. Additional, data in a fragment can only be seen by logged in users.

Realtime

You can listen to any fragment to get changes pushed via websockets. These allows to react to data changes in nearly realtime.

Fake data

With the help of faker.js, a bucket can be filled with fake data. This allows easy testing or scaffolding to get you focused on the work that matters.

Default meta information

Each bucket is enhanced by useful meta information by default. For example data list have paging information attached and details creation and change date.

Access & Pricing

You can access your bucket at any-time with your API-KEY @ https://api.notonly.rest/nr-xxxxxx. Buckets are free, but they might get deleted if a certain data volume is reached.

About

notonly.rest is the cloud to try out the reference implementation of noREST, an not only RESTfull API. It aims to resolve the hassle that comes when implementing an API by providing not only a REST interface, but also a way of storing, authenticate, authorize and realtime accessing. The noREST project is open source and can be found on Github.
